## Project Overview

This is an end-to-end Python pipeline for evaluating AI chatbots using RAGAS (Retrieval-Augmented Generation Assessment). The system generates synthetic test questions from knowledge base documents, collects chatbot responses via browser automation, and evaluates answer quality using RAGAS metrics.

## Core Commands

### Setup
```bash
pip install -r requirements.txt
npx playwright install  # Required for browser automation
```

### Running the Pipeline
```bash
python runner.py         # Full pipeline: generate → collect → evaluate
python3 runner.py        # Alternative on some systems
```

### Running Individual Components
```bash
# Generate synthetic test questions from documents
python scripts/ragas_dataset_generator.py

# Collect chatbot answers via UI automation
python scripts/generate_answers.py

# Evaluate collected answers with RAGAS metrics
python scripts/evaluate_with_ragas.py
```

## Architecture

### Pipeline Flow
1. **Test Generation** (ragas_dataset_generator.py):
   - Loads PDF documents from `DOC_PATH` directory
   - Creates knowledge graph using Azure OpenAI embeddings
   - Applies RAGAS transformations to generate synthetic Q&A pairs
   - Outputs to `generated_testset.json`

2. **Answer Collection** (generate_answers.py):
   - Uses Playwright to automate browser (launches visible browser with `headless=False`)
   - Performs Microsoft authentication (username → password → MFA)
   - Navigates to chatbot URL and asks generated questions
   - Extracts responses from UI elements: `(//div[@class='w-full max-w-full break-words'])[2]`
   - Waits 17 seconds per question for response generation
   - Saves Q&A pairs with ground truth to `data/ragas_dataset.json`

3. **Evaluation** (evaluate_with_ragas.py):
   - Loads dataset from `data/ragas_dataset.json`
   - Evaluates using three RAGAS metrics: answer_relevancy, answer_similarity, answer_correctness
   - Outputs metrics to `ragas_report/metrics_output.json`

### Key Configuration

The pipeline requires two separate Azure OpenAI configurations:

**For test generation** (used by ragas_dataset_generator.py):
- `AZURE_API_KEY`, `AZURE_BASE_URL`
- `AZURE_MODEL_DEPLOYMENT`, `AZURE_MODEL_NAME` (LLM for generation)
- `AZURE_EMBEDDING_DEPLOYMENT`, `AZURE_EMBEDDING_NAME`
- `DOC_PATH` (directory with PDF files for knowledge base)
- `OUTPUT_PATH` (where to save generated testset, default: testset_output.json)
- `SAVE_GRAPH` (optional, saves knowledge graph if "True")

**For evaluation** (used by evaluate_with_ragas.py):
- `AZURE_OPENAI_DEPLOYMENT`, `LLM_BINDING_API_KEY`, `LLM_BINDING_HOST`, `AZURE_OPENAI_API_VERSION`
- `model`, `azure_endpoint`, `api_key`, `api_version`, `deployment` (for embeddings)

**For chatbot access** (used by generate_answers.py):
- `USERNAME1`, `PASSWORD` (Microsoft authentication credentials)
- `BASEURL` (login page URL)
- `CHATBOTURL` (chatbot interface URL)

All configuration is loaded from `.env` file using python-dotenv.

### Directory Structure

- `scripts/` - Core pipeline components (generator, collector, evaluator)
- `config/logger_config.py` - Centralized logging (console + `logs/app.log`)
- `data/` - Input questions, authentication config, output datasets
- `Input files/` - Source PDF documents for knowledge base creation
- `ragas_report/` - Evaluation metrics output
- `screenshots/` - Debug screenshots captured on Playwright errors
- `generated_testset.json` - Synthetic questions generated by RAGAS (root level)
- `knowledge_graph.json` - Knowledge graph if SAVE_GRAPH=True (root level)

### Browser Automation Details

The Playwright automation in `generate_answers.py`:
- Launches Chromium in non-headless mode for visibility
- Performs Microsoft OAuth flow with email → password → submit → MFA button
- Uses explicit waits (`wait_for_load_state('networkidle')`, `wait_for_timeout()`)
- Clicks "Start new chat" button before each question to reset context
- Types questions into placeholder "Type your message here..." and presses Enter
- Waits 17 seconds for response generation (hardcoded timeout)
- Takes full-page screenshots on failures for debugging
- Logs all actions using the centralized logger

### RAGAS Integration

- Uses `TestsetGenerator` with `default_transforms` and `default_query_distribution`
- Knowledge graph built with NodeType.DOCUMENT nodes from PDF page content
- Evaluates with three metrics: answer_relevancy, answer_similarity, answer_correctness
- Dataset format expected by evaluator:
  ```json
  {
    "question": "...",
    "answer": "...",
    "reference": "ground_truth",
    "reference_contexts": [...]
  }
  ```

## Important Notes

- The `runner.py` orchestrates the full pipeline but hardcodes `testset_size=10`
- Logging outputs to both console (INFO level) and `logs/app.log` (DEBUG level)
- Browser automation requires valid Microsoft credentials with MFA capability
- PDF documents in `DOC_PATH` must match the chatbot's knowledge base for accurate evaluation
- The pipeline is configured for Azure DevOps in `ragas_azure_pipeline.yml` (Windows, Python 3.10, Node 18)
